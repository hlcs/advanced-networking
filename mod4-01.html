Network Requirements

Cisco categorized networks based on host number in:

  • Small network - up to 200 devices.
  • Medium network - 200 to 1000 devices.
  • Large network - More than 1000 devices.

Different network sized means different requirements.

It is the job of the network designer to handle the complexities of these requirements and orchestrate connection of various-sized networks.

Cisco has a certification dedicated to network design skills, which encompass a great deal of knowledge. It’s the CCDA, Cisco Certified Design Associate.

[R] Design Good Principles

Hierarchy

Deploying complex systems in smaller and more manageable tiers, each with a clear role and structure.

Modularity

Separating functions in modules (such as Cisco’s campus, service block, data center, internet edge) to make design easier.

Resiliency

Network reacts well to changes, growth, faults, abnormalities.

Flexibility

Expansions, improvements, upgrades at any tier are made as easy as possible, without the need for major operations.

[R] Access, Distribution, Core Layers

[R] Access, Distribution, Core Layers

Tiers In a hyerarchical network architectures are Access, Distrubution and Core, usually deployed in a 3-tier or 2-tier (with collapsed Distribution/Core) framework.

Main advantages: local traffic stays local; better boundary separation (perfect for limiting broadcast or filter traffic).

Traffic starts in the access layer, which is the layer enabling a local network. Traffic can be distributed to different local networks by going through the distribution layer.

If required, a third layer (core) is provided as the backbone, to connect distant networks at speeds similar to the local ones.

[R] Access Layer

  • It's the network edge, where traffic enters or exits the network.
  • It's also where devices that extends the network out one more level are placed - IP phones, wifi APs, WAN connectivity.
  • Its primary function is to provide access to the network for most users (and thus generic computing devices, end stations access here, either from LAN or across a WAN connection.).
  • This is the layer where most traffic originates from.
  • It provides the intelligent demarcation between the network infrastructure and the computing devices that leverage it.
  • The traditional focus at the access layer is minimizing cost-per-port.
  • Summary of its functions: L2 Switching, Spanning Tree, PoE; High Availability; Port Security; QoS Classification; ARP inspection; VLAN ACL, VoIP-dedicated VLANs.

[R] Distribution Layer

  • It's the most "smart" layer, implementing network foundation technologies like routing, advanced switching, QoS and policy-based security (ACL, filtering).
  • Interface between Access and Core Layers.
  • Aggregation point for all access switches (large-scale wiring closet) and WAN links.
  • Aggregating L2 broadcast domains, L3 routing boundaries and summarization boundaries for network control plane protocols (EIGRP, OSPF, Spanning Tree).
  • Demarcation and isolation between the campus distribution building block and the rest of the network.
  • Very little traffic originates directly from here.
  • High availability through redundant distribution layer switches to the end-user and equal cost load balancing to the core.

[R] Core Layer

  • Also known as: network backbone.
  • Very limited set of services, provides 24/7 always-on (Fault Isolation) connectivity across the entire network (end-to-end connectivity).
  • Basically no traffic originates from here.
  • It serves as the aggregator for all of the other campus modules and the traffic coming from all the campus distribution layer switches.
  • Very little processing is done on traffic, just collects as much as possible, moves it as fast and reliably as possible.
  • So: high-speed, high-redundancy forwarding services between "regions" (campus) of the network.
  • That means the most powerful switches and routers, capable of managing the fastest connections (e.g. 10 Gigabit Ethernet).

2-Tier Collapsed Core Design

Sometimes, in small enterprice networks, we can find another design. This design use only two layers with:

  • Access Layer
  • Collapsed Core Layer: a layer with the function of distribution and core.

This design is more cheap than the 3 layer model, saves more hardware (because distribution and core functions are provided in a single device), but mantains much of the benefit of it.

Modular Network Design

In a modular network design, networks are separated into blocks (or modules) depending on their function.

Modules will then have different physical and/or logical connectivity in the overall network design. With modular design:

  • in the event of a failure, the fault is easily isolated and detected in a single module.
  • Upgrades, changes, maintenance of the network can be staged progressively module by module.
  • Modules can be updated or completely replaced without the need to redesign the network, not even a portion of it.

The modular network design reference recommended by Cisco is the Cisco Enterprise Architecture Model, which define several modules and their function.

Cisco Enterprise Architecture Model

Includes the access, distribution and core layers in a Campus Infrastructure Module. The word Campus is common by loosely defined in Cisco’s literature.

[R] Enterprise Campus

A campus is a fixed geographic area in which a network is deployed, but its span can vary: several buildings, an industrial area, the departments of a university. The submodules are:

  • Access - L2/L3 switches providing the needed port density.
  • Access - VLANs, trunking and redundant links to the distribution switches.
  • Distribution - L3 devices to aggregate campus access.
  • Distribution - Routing, QoS, security.
  • Core - High-speed backbone connectivity between.
    • Distribution Modules
    • Server and DataCenter Farms (Network Management happens here)
    • Service Module (IP Telephony, wireless controlles, unified services)
    • Enterprise Edge

[R] Enterprise Edge

All the modules that extends the enterprise network to the outer networks, like remote sites, partner’s Intranets, and of course the Internet.

Obviously, it also enables the campus to provide its services to outer networks. So its submodules are:

  • E-Commerce
    • Networks and servers to provide e-commerce through the Internet. It’s a critical enough activity to deserve its own modules.
  • Internet and Demilitarized Zone (DMZ)
    • It provides internal users with Internet connectivity. It also enabled a DMZ, which is a part of a local network trusted with access from external parties (typically to provide Internet-facing services).
  • WAN modules
    • Devices implementing the WAN technologies in use by the enteprise to connect the campus to remote sites.
  • VPN and Remote Access
    • Authentication and access to the local network for remote users.

[R] Service Provider Edge

Provides the connectivity solutions that power the enterprise edge, in the forms of services from a Service Provider, such as

  • Internet (perhaps multi-homed)
  • PSTN (traditional telephony)
  • Wide Area Networking (Frame Relay, ATM, MetroEthernet...)

All data entering and exiting these connectivity solution passes through a corresponding edge device in the Enterprise Edge.

It is therefore the perfect place to inspect traffic with intrusion detection (IDS) and intrusion prevention (IPS) systems.

Redundancy/failover by multiple connections to ISPs can be:

  • Single-homed - Single connection to a single ISP
  • Dual-homed - 2+ connections to a single ISP
  • Multihomed - Connections to 2+ ISPs
  • Dual-multihomed - Multiple connections to 2+ ISP

Remote Functional Area

The type of areas that might need remote access to the campus local networks are categorized by their functions in:

  • Enterprise Branch
    • A remote site belonging to the enterprise that allows employees to work on campus resources from a non-campus location.
    • Enabling edge devices can be WAN interfaces, or Internet+VPN as a backup.
  • Enterprise Teleworker
    • It enables to provide connectivity to employees that want or need to work from disperse locations, including non-enterprise ones such as homes, hotel, customer sites.
    • To enable edge devices: mostly local ISP connection + VPN.

Remote Functional Area

  • Enterprise Data Center
    • Same functions as campus data center, but in a remote location.
    • Enabling edge devices are mostly fast WAN services like MetroEthernet, enabling (a)synchronous data+app replication.

Cisco Enterprise Architecture

New technologies as

    • Cloud
    • Internet of the things (IoT)
    • Online collaboration
    • Bring Your Own Device (BYOD)

    were a great challenge for network designer, because they make the border between networks’ components, areas, responsibilities and policies less clear.

    Use scenarios change along with new technologies and the designer works to implement them without disrupting existing networks. To help designers Cisco has introduced 3 new network architectures

    • Cisco Borderless Network Architecture
    • Collaboration Architecture
    • Data Center/Virtualization Architecture

Borderless Network Architecture

It’s a complete solution by Cisco to armonize the use of personal devices in the enterprise network.

It enables IT departments to configure personal devices in a seamless fashion, to bring them in compliance with the network policies while enabling employees to work with them.

A practical example: how can personal devices such as laptop, tablets, phones, with many different operating system, can be easily configured to connect securely and easily to the enterprise network?

The Cisco’s answer is using a multiplatform software developed for these end-devices, called Cisco AnyConnect.

Collaboration Architecture

Groupware are software applications that are meant to be used by more than one user to achieve their purpose. Applications that let people collaborate to produce something (think of Gooogle Doc) are an example.

Cisco has an architecture to enable the creation and implementation of collaboration application, made by 3 layers:

Collaboration Architecture

  • Ready to use applications and devices
    • Cisco WebEx Meetings/ Social, Jabber and a popular teleconference hardware line, TelePresence. They include voice, video, messaging, enterprise social networks, complete with mobile versions.
  • Collaboration Services
    • API and SDK that allows developer to build custom application and features based on Cisco collaboration tech providing services such as contact, location, policy and session management, tagging, etc.
  • Network and Computer Infrastructure
    • Hardware capable of powering always-on collaboration tools, such as virtual machines, storage systems, high-performance networks.

DataCenter/Virtualization Arch.

  • Cisco Unified Management Solutions
    • Automation for resource (both virtual and physical) management
    • Provisioning and orchestration
  • Unified Fabric Solutions
    • Switches (Catalyst and Nexus) and their operating system (Cisco NX-OS, Linux-based), specific for data center operations.
    • It replaces many I/O systems on servers with a single high-bandwidth and low-latency network cable, simplifying data center architecture and improving resource usage and sharing.
  • Unified Computing Solutions
    • Cisco’s server solutions, designed to lower the TCO (Total Cost of Ownership) though virtualization technologies.
