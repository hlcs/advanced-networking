DHCPv4 Allocation

Dynamic Host Configuration Protocol is the most used method for providing addressing informations to final users.

DHCP is a client-server protocol. Cisco routers can act as both DHCP servers and clients. There is also an optional feature set called Easy IP that enables a more advanced DHCP server.

DHCPv4 allows for 3 different allocation mechanisms:

  • Manual - A pre-allocated address is configured by the administrator to be always assigned to a specifc device. DHCP has only the task to communicate it to the device.
  • Automatic - The server selects an address from the pool, and permanently reserves it to that device.
  • Dynamic - The server selects an address from the pool and leases it to a device for a limited (configurable) time, after which the address is returned and the client must renew it.

DHCPv4 Operation

DHCPv4 Message Format

This format is used for all DHCP messages and encapsulated in UDP (although several DHCP implementations can also use TCP) with 67 as the client port and 68 as the server port.

DHCPv4 Message Format

  • Operation Code - The message is a 1 = request; 2 = reply.
  • Hardware Type - It indicates the type of network: Ethernet, Frame Relay, etc.
  • HW Address Length - Length of the L2 addresses in use.
  • Hops - Set to 0 by a client before transmitting a request and used for relay.
  • Transaction Identifier - 32 bits number generated by client to match requests and replies in DHCP transaction.
  • Seconds - Time since a client began an attempt to acquire or renew a lease. It’s used by the server to prioritize replies.
  • Flags - The first bit is set to 1 by the client when it doesn’t have an IP to inform that replies should be sent as broadcast.
  • Client IP Address - Until the client is missing a valid IP, is set to 0. On renewal, the client inserts its valid current IP.

DHCPv4 Message Format

  • Your IP Address - The server use it to deliver the allocated IP.
  • Server IP Address - Used by server to inform the client of the next server to use in the DHCP transaction (not necessarily the one sending the current reply).
  • Gateway IP Address - Used in relay operations.
  • Client Hardware Address - L2 address of the client.
  • Server Name - The server has the option to insert its name here, either an hostname or a FQDN.
  • Boot Filename - Used by clients (to request) and by servers (to provide) the path to a particular bootfile.
  • DHCP Options - Variable-length field used for storing several parameters, can be used by both client and server. For instance, the sending server can always identify itself through the Server Identifier DHCP option, even when it’s a relay agent.

DHCDISCOVER Messages

  • Source MAC - MAC address of the client
  • Destination MAC - FF:FF:FF:FF:FF:FF
  • Source IP - 0.0.0.0
  • Destination IP - 255.255.255.255
  • Destination Port - 67
  • CIADDR - 0.0.0.0
  • GIADDR - 0.0.0.0
  • CHADDR - MAC address of the client
  • Mask - 0.0.0.0

DHCPOFFER Messages

  • Source MAC - MAC address of the server
  • Destination MAC - MAC address of the client
  • Source IP - IP of the server
  • Destination IP - IP the server is leasing to the client.
  • Destination Port - 68
  • CIADDR - IP the server is leasing to the client.
  • GIADDR - 0.0.0.0
  • CHADDR - MAC address of the client
  • Mask - The appropriate subnet mask for this network.

Configure DHCPv4 Server

  • Exclude from the pool the IP addresses that are statically assigned to certain hosts, so the DHCP server will avoid assigning those and causing a conflict.
    • Router(config)# ip dhcp excluded-address {low_address} [high_address]
  • Create the DHCP pool and enter in DHCP config mode
    • Router(config)# ip dhcp pool {pool_name}
Router(dhcp-config)#
  • Add the desired address range in the pool
    • Router(dhcp-config)# network {network_address} [mask | prefix-length]
  • Add the default gateway
    • Router(dhcp-config)# default-router {address} [address_2 ... address_8]

Configure DHCPv4 Server

  • The following steps are optional:
    • Router(dhcp-config)# dns-server {address} [address_2 ... address_8]
Router(dhcp-config)# domain-name {domain}
Router(dhcp-config)# lease {days [hours] [minutes] | infinite}
Router(dhcp-config)# netbios-name-server {address} [address_2 ... address_8]
  • To disable DHCPv4
    • Router(config)# no service dhcp
  • To (re)enable it (it first must be configured)
    • Router(config)# service dhcp

DHCPv4 Relay

In modern networks, servers are typically assigned their own, separate subnet. This is a problem for services relying on broadcast-based protocols like DHCP does.

If the DHCP server is on a different subnet, it won’t be able to receive the broadcast requests sent by clients on a different subnet, even if it’s configured to handle those pools.

The solution is to set one or more router as DHCP relay agents.

DHCPv4 Relay

DHCP relay agents will:

  • take broadcast requests from from clients on their subnet and forward them to either the actual DHCP server or another relay agent.
  • take replies coming from the actual DHCP server and forward them to the client that initiated that transaction.

A Cisco router can be configured as DHCPv4 relay agent (but also for other protocols such as DNS) by entering the ip helper-address [server_ip] command on an interface: the router will forward DHCP requests coming from client on that interface.

The helper address will show up in the show ip interface [id] output.

Cisco Router as DHCPv4 Client

Even if configured as DHCPv4 server to handle private IP addresses for local networks, it's usual for routers that manages a direct connection to the Internet to act also as DHCP clients.

Unless we bought a service with static public IP addresses, it will receive a dynamic public IP at every connection.

DHCPv4 Troubleshooting

  • Solve any address conflict.
    • The show ip dhcp conflict command will show conflict detected by the server through ARP and ICMP:
      •  Router# show ip dhcp conflict

IP address Detection Method Detection time

192.168.10.32   Ping            Feb 18 2014 13:35 PM
192.168.10.64   Gratuitous ARP  Feb 22 2014 07:01 AM
  • Check physical connectivity.
    • DHCP will never ever work if interfaces are not in up/up.
  • Test a static IPv4 config.
    • If the connectivity issues remain after statically assigning an IP address on the client, then it's not DHCP's fault.
  • Is DHCP disabled?
    • Check for no service dhcp in the running config.

DHCPv4 Troubleshooting

  • Startup negotiation failure
    • If on boot the client wasn't able to acquire an IP address, try to manually initiate a new DHCP transaction.
  • Check for possible relaying issues.
    • Check if the issue vanishes if the client is moved in the same subnet/VLAN as the server.
    • Is the ip helper-address configuration correct? It must be present on the inbound interface of the LAN containing the DHCPv4 client and must be directed to the correct DHCPv4 server.
  • Turn on DHCP debugging
    • debug ip dhcp server events shows address assignments, database updates, renewals,etc.

Stateless Address AutoConfiguration

SLAAC is a method to obtain an IPv6 global unicast address without relying on a DHCPv6 server.

SLAAC relies on ICMPv6 which is similar to ICMPv4 but has more functionalities and is a much more robust protocol.

SLAAC uses ICMPv6 Router Solicitation and Router Advertisement messages to provide informations needed to autoconfigure IPv6 addresses with no need for a DHCPv6 server.

This process is called stateless because the server is not maintaining any client state information, it only provides configuration parameters for clients, not IPv6 addresses.

Router Solicitation and Advertisement

  • Router Solicitation (RS) messages
    • the client sends an RS message to the router using the IPv6 all-routers multicast address FF02::2 as destination.
  • Router Advertisement (RA)messages
    • Sent by routers to clients to provide addressing info.
    • A router sends an RA message periodically, or in response to an RS message.
    • Always sent to the IPv6 all-nodes multicast address FF02::1.
    • The RA message includes the prefix and prefix length of the local segment.
    • A client uses this information to create its own IPv6 global unicast address.
    • By default, Cisco routers send RA messages every 200s.

SLAAC IP generation on clients

After receiving a RA message the host now has a 64-bit network prefix, but needs a 64-bit Interface ID (IID) to create a global unicast address.

There are two ways the host can create its own unique IID:

  • EUI-64 - Using the EUI-64 process, the host will create an IID using its 48-bit MAC address.
  • Randomly generated - The 64-bit IID can be a random number generated by the client operating system.

After generating the global unicast address, the host has no guarantee that it's unique (even though a conflict is higly unlikely) because of the stateless process.

It use an ICMP Neighbour Solicitation directed to its newly generated IP. If no reply is heard, the assignment is completed.

SLAAC and DHCPv6

ICMPv6 RA messages contain two flags that suggest to the client which address allocation solution it should use.

  • The Managed Address Configuration flag (M flag)
  • The Other Configuration flag (O flag)

Using different combinations of the M and O flags, RA messages signals one of three addressing options for the IPv6 client:

  • SLAAC (Router Advertisement only) - M = 0, O = 0
  • Stateless DHCPv6 (Router Advertisement and DHCPv6) - M = 0, O = 1
  • Stateful DHCPv6 (DHCPv6 only) - M = 1

SLAAC-only

With SLAAC enabled, the router is telling the client to rely on the content of the RA for everything: prefix, prefix-length, DNS server and gateway addresses, MTU.

The RA as both the M flag and the O flag set to 0.

SLAAC is the default method configured on Cisco routers.

To manually enable SLAAC for an interface the M and O flags need to be reset to their initial values of 0. This is done using the following interface configuration mode commands:

Router(config-if)# no ipv6 nd managed-config-flag
Router(config-if)# no ipv6 nd other-config-flag

Stateless DHCPv6

In a steteless DHCPv6 setup, the router is telling the client to use SLAAC for allocating a global unicast address, but that additional information is available through a DHCPv6 server.

The DHCPv6 can provide DNS servers, gateway addresses,etc.

As SLAAC, it is stateless because the DHCPv6 server is only providing RAs and additional infos, but still doesn't keep track of the IPv6 addresses.

The RA has the M flag set to 0 and the O flag set to 1.

To configure a router to advertise a stateless DHCPv6 setup:

Router(config-if)# ipv6 nd other-config-flag

Stateful DHCPv6

This is basically identical to how a DHCPv4 server operates. The router is telling the clients that it will manage all the addressing information as well as additional parameters.

Stateful means that the DHCPv6 server will manage and keep track of the addressing informations handed to the clients.

The M flag set to 1 in the RA advertise a DHCPv6 Stateful setup. The O flag is irrelevant.

DHCPv6 Operations

DHCPv6 client/server communication are initiated by a (periodic or solicited) ICMPv6 RA message from the router, indicating stateless or stateful setups.

DHCPv6 server-client messages uses UDP dst-port 546.

DHCPv6 client-server messages uses UDP dst-port 547.

  • In order to locate the server, the client sends a DHCPv6 SOLICIT message to the reserved IPv6 multicast all-DHCPv6-servers address FF02::1:2 (which has link-local scope).
  • Server(s) respond with a DHCPv6 ADVERTISE message.
  • The client responds to the server with a:
    • DHCPv6 REQUEST message, in stateful DHCPv6.
    • INFORMATION-REQUEST message, in stateless setups.
  • The server sends a DHCPv6 REPLY to the client containing the requested informations.

Configure Stateless DHCPv6 Server

## ENABLE IPV6 ROUTING
## Not required for operating as DHCPv6 server but
## Needed to be able to send RA messages
Router(config)#ipv6 unicast-routing 
## CREATE THE POOL + ENTER DHCPv6 CONFIGURATION MODE
Router(config)#ipv6 dhcp pool STATELESS-V6 
## CONFIGURE POOL PARAMETERS
Router(config-dhcpv6)#dns-server 2001:db8:abcd:1234::fffe
Router(config-dhcpv6)#domain-name hlcs.it
Router(config-dhcpv6)#exit
## BIND THE DHCPV6 POOL TO THE INTERFACE
Router(config)#interface gigabitethernet0/0
Router(config-if)#ipv6 address 2001:db8:abcd:1::1/64
Router(config-if)#ipv6 dhcp server STATELESS-V6
Router(config-if)#ipv6 nd other-config-flag

Configure Stateful DHCPv6 Server

Very few differences to stateful configuration:

Router(config)#ipv6 unicast-routing
Router(config)#ipv6 dhcp pool STATEFULL-V6
## THIS TIME, ADDRESSING INFORMATION IS CONFIGURED FOR THE POOL
## SYNTAX: adress prefix {prefix-length} [lifetime {valid_lifetime preferred-lifetime | infinite}
Router(config-dhcpv6)#address prefix 2001:db8:abcd:1::/64 lifetime infinite
Router(config-dhcpv6)#dns-server 2001:db8:abcd:1234::fffe
Router(config-dhcpv6)#domain-name hlcs.it
Router(config-dhcpv6)#exit
Router(config)#interface gigabitethernet0/0
Router(config-if)#ipv6 address 2001:db8:abcd:1::1/64
Router(config-if)#ipv6 dhcp server STATEFULL-V6
## WE NEED TO ADVERTISE THE STATEFUL SETUP
Router(config-if)#ipv6 nd managed-config-flag

Cisco Router as DHCPv6 Client

The router needs an IPv6 link-local address on the interface to send and receive RA, RS and DHCPv6 messages.

The link-local address is created automatically when an IPv6 global unicast address is assigned, or by manually enabling IPv6 with the command ipv6 enable.

  • To configure the router as client in a stateless setup:
    • Router(config)#interface gigabitethernet0/0
Router(config-if)#ipv6 enable
Router(config-if)#ipv6 address autoconfig
  • As client in a stateful setup:
    • Router(config)#interface gigabitethernet0/0
Router(config-if)#ipv6 enable
Router(config-if)#ipv6 address dhcp

Cisco Router as DHCPv6 Relay Agent

When a router on a segment is configured as DHCPv6 relay, client will send their messages to the IPv6 multicast All DHCPv6 Relay Agents and Servers address FF01::1:2.

Router(config)#interface gigabitethernet0/0
## THIS IS THE EQUIVALENT OF IPv4 ip helper-address COMMAND
Router(config-if)#ipv6 dhcp relay destination 2001:db8:abcd:1::f
Router(config-dhcpv6)#end
Router#show ipv6 dhcp interface gigabitethernet0/0
GigabitEthernet0/0 is in relay mode
     Relay destinations:
       2001:db8:abcd:1::f
Router#
