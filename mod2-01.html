Hyerarchical Network Design

Cisco has a design guideline for network architecture called Cisco Borderless Network. Its principles are:

  • Hierarchy - Deploying complex systems in tiers, each with a clear role and structure.
  • Modularity - Expansions, improvements, upgrades at any of these tiers are made as easy as possible.
  • Resiliency - Network reacts well to changes, growth, faults.
  • Flexibility - The network load is intelligently splitted to make the best use of resources.

Time has proved that the best way to achieve the last 3 is through a hierarchical network architecture.

Access, Distribution, Core

Tiers In a Cisco network architecture are Access, Distrubution and Core, usually deployed in a 3-tier or 2-tier (with collapsed Distribution/Core) framework.

Access Layer

  • It's the network edge, where traffic enters or exits it.
  • It's also where devices that extend the network out one more level are placed - like IP phones, wireless APs, WAN connectivity.
  • It's primary function is to provide access to the network for most users (and thus generic computing devices, end stations access here).
  • This is the layer where most traffic originates from.
  • It provides the demarcation between the network infrastructure and the computing devices that leverage it.
  • The traditional focus at the access layer is minimizing cost-per-port.

Distribution Layer

  • It's the most "smart" layer, implementing network foundation technologies like routing, advanced switching, filtering, QoS.
  • Interface between Access and Core Layers:
    • Aggregation point for access switches (wiring closet).
    • Aggregating L2 broadcast domains and L3 routing and summarization boundaries for network control plane protocols (EIGRP, OSPF, Spanning Tree).
    • Demarcation and isolation between the campus distribution building block and the rest of the network.
  • Very little traffic originates directly from here.
  • High availability through redundant distribution layer switches and equal cost paths to the core.

Core Layer (Backbone)

  • Very limited set of services, provides 24/7 always-on (Fault Isolation) connectivity across the entire network (end-to-end connectivity).
  • Basically no traffic originates from here.
  • It serves as the aggregator for the other campus blocks and ties the campus together with the rest of the network.
  • Very little decisions about traffic, just collects as much as possible, moves it as fast and reliably as possible.
  • So: high-speed, high-redundancy forwarding services between "regions" (campus) of the network.
  • That means the most powerful switches and routers, capable of the fastest connections (e.g. 10 GiB Ethernet)

Choosing a Switch

The typical parameters for choosing a switch are:

  • Cost: dependent by the trade-off involving all other parameters in function of the requirements.
  • Port Density: number of ports available.
  • Port Speed: network connection speed to users.
  • Backplane: the maximum switching capacity.
  • Power: Redundant power supply, PoE.
  • Frame Buffers: the amount of frames that the switch can store before it must forward or discard them.
  • Scalability: the ability of the switch to support network growth.
  • Additional Features: like L3 supports, USB ports, etc.

Switch Form Factor

Configuration options representing the switch form factor:

  • Fixed - These are switches that cannot be expanded with additional ports or port types.
  • Modular - Chassis with free slots for installation of modular line cards that expand ports number/type.
  • Stackable - Switches that can be interconnected with a high-bandwidth special cable, effectively operating as a single logical switch.
    • Cisco Stackwise: up to 9 switches.
    • Cross-connected for fault tolerance.
    • Cisco StackPower for sharing PSUs.

Switching Fundamentals

Switching is the process through which a device makes a decision about where to forward network traffic. It is based on the ingress port and the destination address.

This is true regardless of the port (technology) and address (protocol) type. Switching is a telecommunication concept.

  • Ingress - The spot where a frame enters a device.
  • Egress - The spot where a frame leaves a device.
  • We'll typically talk about ingress port(s) and egress port(s).

A LAN Switch, the type we'll focus on, has the ability to build a table based on Ethernet MAC Addresses, called MAC Table, which we'll be used to make switching decisions.

CAM Table

Switches are made by dedicated-purposes motherboards (ASICs) that can internally move large amount of received frames at very high speed.

The switch has to build a MAC Table to make its decisions.

It's also called CAM table because of the specialized (content addressable) memory in which switches stores it.

  • The MAC table associates each port with MAC addresses.
  • It is populated by examining source MAC adresses.

Frame Forwarding - Ingress

  • The MAC table is empty when the switch is first powered on.
  • When a frame is received, it looks at the source MAC address.
    • If it's not on the table, an association is added between the source MAC and the ingress port the frame appeared on.
    • If it was already on the table, the aging timer of the association is reset. Maximum aging time is usually 5 min.

Frame Forwarding - Egress

  • Then, it looks at the destination MAC address
    • If It's not on the table it forwards the frame to all ports except the ingress port. This is called flooding.
      • Only the destination will reply with a unicast frame. The switch can now add the missing association in the table.
    • If It's already on the table, the switch forwards the frame to the associated egress port.
  • Once all the associations have been learned the switch doesn't need flooding anymore.

Store & Fordward Switching

Store and Forward Switching makes a forwarding decision only after receiving the entirety of the frame and error-checking it.

It is the main switching method in a Cisco LAN switch.

After receiving the frame, the switch calculate its checksum (FCS). If it matches the frame's FCS value then it's forwarded through an egress port, otherwise it's dropped.

S&F switching could be problematic if the switch ingress and egress ports are operating at different speeds. That's why the switch automatically sets up an appropriate buffer.

Cut-Through Switching

Cut-Through Switching makes a forwarding decision as soon as "enough" bytes of the frame as been received.

Thus with C-T switching the actual forwarding process begins before the frame has been completely received.

There are two variants of cut-through switching, and both of them do not perform a checksum verification:

  • Rapid Frame - The switch forwards the frame as soon as the destination MAC address (first 6 bytes) is received.
  • Fragment Free - The switch waits for the first 64 bytes (collision window). If no fragmentation is detected it then forwards the frame.

Switching Methods Comparison

  • With Store & Forward switching invalid frames are never forwarded.
  • Cut-Through switching forwards almost every invalid frame.
  • Cut-through provides significantly better latencies.
  • In a network with a high error rate, C-T switching becomes counterproductive, wasting bandwidth with invalid frames.
  • Fragment-Free is usually the best compromise: a little better error checking with virtually no increase in latency.

Collision Domain

A collision domain is a portion (segment) of the network where frames can physically collide if transmitted at the same time, because they share a common medium and its bandwidth.

Hubs constitute a single collision domain: a frame is forwarded on every port, making the medium unavailable to other transmissions in any direction.

A switch is capable (though microsegmentation) of reducing or avoiding the competition on a medium: either each port becomes its own collision domain (half-duplex links) or collisions are impossible (full-duplex).

Broadcast Domain

When a switch receives an Ethernet broadcast frame (all-1s source MAC), it floods it on every port. Other connected switches will forward the frame in the same way.

These behaviour allows to define the broadcast domain: the portion of a network in which devices can reach each other through broadcast frames at the data link layer.

They are said to see each other at Layer 2.

Broadcast Domain

Each interconnected switch increases the broadcast domain. A network layer/L3 device such as a router must be used to put boundaries on (segment) a broadcast domain.

Broadcast is necessary, but due to the way it propagates it can quickly produce a lot of traffic and slow-downs. A broadcast domain control strategy is essential.
