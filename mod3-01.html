Enteprise Networks

The order and magnitude of the problems that an enteprise network faces are very different from the small, home or office networks we are used to.

Namely scalability and high availability.

An enterprise networks runs more services, users and locations.

The concept of critical services, and the needs for converged traffic and central administrative control are introduced.

Five Nines

Enterprise networks are expected to have an uptime of five-nines, 99.999%, which translates into 5 minutes of downtime per year.

The difference between 2 nines and 5 nines is 3 days!

To achieve this, hardware that complies to higher standards is required, increasing costs.

A common feature found in enterprise hardware is components failover.

Hierarchical Networks

Designing a network hierarchy aid in optimizing the traffic flow.

Traffic starts in the access layer, which is the layer enabling a local network. Traffic can be distributed to different local networks by going through the distribution layer.

If required, a third layer (core) is provided as the backbone, to connect distant networks at speeds similar to the local ones.

Cisco Enterprise Network Architecture

On top of the hierarchical structure, Cisco defines an architecture for enteprise networks based on functional blocks that is called Cisco Enterprise Architecture.

Enterprise Campus

Includes the access, distribution and core layers in a Campus Infrastructure Module.

  • Access - L2/L3 switches providing the needed port density.
  • Access - VLANs, trunking and redundant links to the distribution switches.
  • Distribution - L3 devices to aggregate campus access
  • Distribution - Routing, QoS, security
  • Core - High-speed backbone connectivity between
    • Distribution Modules
    • Server and DataCenter Farms (Network Management takes place here)
    • Service Module (IP Telephony, wireless controlles, unified services)
    • Enterprise Edge

Enterprise Edge

All the modules that extends the enterprise network to the outer networks, like remote sites, partner’s Intranets, and of course the Internet. So:

  • Internet modules
  • WAN modules
  • VPN modules

It provides QoS, security, and policy enforcement.

Service Provider Edge

Provides the connectivity solutions that power the enterprise edge, such as

  • Internet (perhaps multi-homed, which means connection to 2 or more ISPs)
  • PSTN (traditional telephony)
  • Wide Area Networking (Frame Relay, ATM, MetroEthernet...)

All data entering and exiting these connectivity solution passes through a corresponding edge device in the Enterprise Edge.

It is therefore the perfect place to insepect traffic with intrusion detection (IDS) and intrusion prevention (IPS) systems.

Failure Domains

A failure domain is the area of a network that is impacted by a failure in a component, device, service or system.

Good network designs limit failure domains as much as possible.

Failure Domains

The failure domains is mostly determined by the function of the device: a failing L2 switch affects only a specific network segment, while a faulty router affects internetwork connectivy.

The reliability standard and features (redundant components, stackability, modularity, link aggregation, etc) are one of the main things separating enterprise-class devices from the consumer’s equivalent (as reflected by costs).

Providing redundancy at the core layer is highly difficult and expensive, so most designs focus on the distribution layer.

  • L3 devices acting as gateways for a limited number of users limits the failure domain.
  • L3 devices are deployed in pairs (switch block), with redundant links to access devices splitted evenly between those.

Scalability Recommendations

Scalability is the capability of a network to accomodate growth.

  • Use expandable, modular, stackable devices.
  • Hierarchical network design.
  • Hierarchical addressing strategy.
  • Choose L3 devices as much as possible to be able to filter traffic as needed, especially broadcast.
  • Implement redundant links between the most critical devices.
  • Increase bandwidth using link aggregation technologies like EtherChannel (with significant savings compared to higher speed interfaces).
  • Wireless is an easy way to expand a network.
  • Implement a routing protocol that is scalable, like a link-state routing protocol.

Redundancy

Redundancy

Redundancy is the deployment of duplicate equipment to avoid single point of failure situations. It allows for failover, which is the automatic switch from a faulty system to a backup one.

Duplicate equipment calls for redundant links between devices, which can result in L2 switching loops (a frame starts circulating endlessly between these multiple paths) and broadcast storms.

Switching loops are prevented by a protocol called Spanning Tree Protocol (STP) which is required whenever multiple paths between switches are provided.

It’s a mechanism that disables redundant paths until they are needed. In any moment a single path to an L2 endpoint is active.

Uplink and Link Aggregation

An uplink is a single (outgoing) link into which traffic coming from several links converges into. Uplinks are common spots in hierarchical networks where bottlenecks can occur.

Having to collect traffic coming from a multitude of sources, uplinks must have a greater bandwidth at their disposal.

Instead of using faster links, a technology called link aggregation can be used by combining several, already existing, physical links into a single logical one.

EtherChannel

EtherChannel is a Cisco technology for link aggregation that combines multiple switch ports (anf their links) into a single EtherChannel interface (and link).

An EtherChannel interface operates at the combined bandwidth of all the ports involved, and can load balance traffic across the physical links.

Routing in the Enterprise Network

Enteprise networks are comprised of many local networks dispersed in several locations, and as such require scalable routing protocols to handle end-to-end connectivity.

OSPF (link-state) and EIGRP (distance-vector) are two popular routing protocols used in Cisco’s L3 devices.

Both protocols have several settings to adapt to a range of situations. OSPF, for instance, can be used first in single-area mode, and then expanded into a multi-area configuration as the enterprise network expands.

EIGRP is an advanced distance-vector protocol with it’s well supported on Cisco devices and heavily used in networks which deploy mostly Cisco equipment.

Cisco Switch Platforms

Cisco has a few switch categories for enteprise networks.

  • Campus LAN
  • Cloud-Managed
  • Data Center
  • Service Provider
  • Virtual Networking

Port Density

Port density is the number of ports that are presents on a switch. Higher density switches optimize the use of power and space.

Remember that some ports (uplinks) are "wasted" because they are used to connect switches to the network infrastructure and thus are not available for users.

Switches may also have SFP (Small Form-factor Pluggable) ports to be used for installing faster interfaces for the uplinks. Another option, popular on fixed configuration switches, is link aggregation.

Modular switches can provide even more than 1000 ports, an order of magnitude higher than fixed configuration switches.

Forwarding Rate

A fundamental feature of every switch is its forwarding rate, which is the amount of data it can process, per second.

Enterprise switches are able to switch data at full wire speed (the maximum possible bandwidth for a link) for all the ports simultaneously.

Not every switch has to or could work at full wire speed, for instance because they are limited by their uplinks anyway. This is the typical situation of an access switch.

Cisco Routers

  • Branch Routers (for local routing and WAN access).
  • Network Edge Routers (boundary routers designed for multi-WAN deployment).
  • Service Provider Routers (designed for Service providers who wants to differentiate their services).
