LLC and MAC

Ethernet is the most widely used LAN technology used today

Ethernet operates in the data link layer and the physical layer

IEEE 802.2 and 802.3 standards

LLC sublayer

  • handles the communication between the upper layers and the lower layers and adds control information to help deliver the packet to the destination node
  • takes the network protocol data

MAC sublayer

  • MAC is implemented by hardware
  • MAC constitutes the lower sublayer of the data link layer

MAC

Ethernet MAC sublayer has two primary responsibilities:

  • Data encapsulation
  • Media access control

Data encapsulation

  • Data encapsulation
  • Addressing
  • Error detection

Media Access Control

  • Is responsible for the placement of frames on the media and the removal of frames from the media
  • Carrier Sense Multiple Access

CSMA

Is used to first detect if the media is carrying a signal

Do not require mechanisms for tracking whose turn it is to access the media

As use and the number of nodes increases, the probability of successful media access without a collision decreases

CSMA/Collision Detection

  • The device monitors the media for the presence of a data signal.
  • If a data signal is absent the device transmits the data

CSMA/Collision Avoidance

  • Used in wireless
  • The device examines the media for the presence of a data signal
  • If the media is free, the device sends a notification across the media of its intent to use it

Ethernet Frame

Early versions of Ethernet were relatively slow at 10 Mbps. The latest versions of Ethernet operate at 10 Gigabits/s

Both the Ethernet II and IEEE 802.3 standards define the minimum frame size as 64 bytes and the maximum as 1518 bytes

Any frame less than 64 bytes is considered a collision fragment

The IEEE 802.3ac extended the maximum allowable frame size to 1522 bytes for Virtual Local Area Network (VLAN)

MAC Address

Is Hexadecimal

Ipconfig, ifconfig, ip

Unicast address and Broadcast address (FF-FF-FF-FF-FF-FF)

Multicast Address: allow a source device to send a packet to a group of devices

Starts with 01-00-5E

MAC and IP

Physical address (the MAC address)

Logical address (the IP address)

On Ethernet networks, MAC addresses are used to identify, at a lower level, the source and destination hosts

IP addressing determines the end-to-end behavior of an IP packet.

Address Resolution Protocol

Each node on an IP network has both a MAC and an IP address

The ARP protocol provides two basic functions:

  • Resolving IPv4 addresses to MAC addresses
  • Maintaining a table of mappings

When a node do when it needs to create a frame and the ARP cache does not contain a map of an IP address to a destination MAC address generates an ARP request!

If the destination IPv4 host is not on the local network, the source node needs to deliver the frame to the router interface that is the gateway or next hop used to reach that destination

ARP cache timer removes ARP entries that have not been used for a specified period of time

On a Cisco router, the show ip arp command is used to display the ARP table

ARP Issues

  • Overhead on the Media
  • Security
  • Broadcast and security issues related to ARP can be mitigated with modern switches
  • Switches provide segmentation of a LAN, dividing the LAN into independent collision domains

Switching

A Layer 2 LAN switch performs switching and filtering based only on the OSI data link layer (Layer 2) MAC address

Layer 2 switches depend on routers to pass data between independent IP subnetworks.

A switch determines how to handle incoming data frames by using its MAC address table.

Switch builds its MAC address table by recording the MAC addresses of the nodes connected to each of its ports

Duplex Settings

Half Duplex

  • communication relies on unidirectional data flow where sending and receiving data are not performed at the same time
  • communication relies on unidirectional data flow where sending and receiving data are not performed at the same time

Full Duplex

  • data flow is bidirectional, so data can be sent and received at the same time
  • In full-duplex mode, the collision detect circuit is disabled

Forwarding Methods

Store-and-forward switching

  • when the switch receives the frame, it stores the data in buffers until the complete frame has been received
  • the switch also performs an error check using the Cyclic Redundancy Check (CRC) trailer portion of the Ethernet frame.

Cut-through switching

  • the switch acts upon the data as soon as it is received, even if the transmission is not complete
  • Fast-forward switching
    • Fast-forward switching immediately forwards a packet after reading the destination address
  • Fragment-free switching
    • the switch stores the first 64 bytes of the frame before forwarding

Memory Buffering on Switches

Switch may use a buffering technique to store frames before forwarding them

Buffering may also be used when the destination port is busy due to congestion and the switch stores the frame until it can be transmitted

Port-based Memory Buffering: frames are stored in queues that are linked to specific incoming and outgoing ports

Shared Memory Buffering: deposits all frames into a common memory buffer that all the ports on the switch share

Switch Configuration

Fixed Configuration Switches

Cannot add features or options to the switch beyond those that originally came with the switch

Modular Switches

More flexibility in their configuration

Layer 2 vs Layer 3 Switching

Layer 2 LAN switch

  • performs switching and filtering based only on the OSI data link layer (Layer 2) MAC address
  • depends upon routers to pass data between independent IP subnetworks

Layer 3 switches

  • are also capable of performing Layer 3 routing functions
  • reducing the need for dedicated routers on a LAN

Cisco Express Forwardings

Decouples the usual strict interdependence between Layer 2 and Layer 3 decision making

The two main components of CEF operation are

  • Forwarding Information Base (FIB)
    • FIB is conceptually similar to a routing table
    • FIB is updated when changes occur in the network and contains all routes known at the time
  • Adjacency tables
    • maintain Layer 2 next-hop addresses for all FIB entries

Type of Layer 3 Interfaces

Switch Virtual Interface (SVI)

Logical interface on a switch associated with a virtual local area network (VLAN)

Routed Port

Physical port on a Layer 3 switch configured to act as a router port.

Layer 3 EtherChannel

Logical interface on a Cisco device associated with a bundle of routed ports
