Operating Systems

An OS is the interface between the hardware and the user.

Its main components are a kernel, which directly manages the hardware, and a shell, which is the interface provided to the user.

A shell can be graphical (GUI, Graphical User Interface) or text-based (CLI, Command-Line Interface).

The OS on embedded devices is usually called firmware. A firmware managing the functions of networking devices such as switches or routers is a network OS.

GUI vs CLI

GUIs are, of course, much easier and user-friendly for the average user. It requires less knowledge.

A network technician, administrator or designer it's not the average user :-). CLIs provides tangible advantages for expert users:

  • They are a standard, well-documented way of interacting with the OS, while GUIs have more variety.
  • A CLI is very lightweight, while a GUI it's a much more complex software (thus almost always less stable and slower).
  • GUIs don't always expose all of the feature set, while CLIs provide access to everything configurable.

Cisco IOS

Cisco IOS (Internetwork Operating System) is a term that encompasses a number of different OSes that run on various networking devices.

There are many distinct variations of Cisco IOS:

  • IOS for switches, routers, and other Cisco networking devices.
  • IOS numbered versions for a given Cisco networking device.
  • IOS feature sets providing distinct packages of features and services.

In this course, we'll mainly focus on Cisco IOS Release 15.x.

GNU/Linux Operating System

Gnu/Linux is a general purpose free operating system based on Linux Kernel.

The are other free operating systems based on many flavours of BSD kernel.

Different packaged OS are called distributions, like Fedora, Red Hat, Ubuntu, Debian, Arch and many more.

Some are for networking purpose, like: IPCop, PFSense, ZeroShell, CertOS, OpenWRT and many more.

In this course, you will focus on general purpose free OS.

Accessing an IOS Device

There are several ways to access the CLI environment: through the console and AUX ports (out-of-band), or through the Telnet and SSH services (in-band).

The console port is a management port that provides out-of-band access, which means access via a dedicated channel used just for device maintenance.

It's used for performing the initial configuration and for recovering purposes, because it's always available even when there's no network access to the device.

The AUX is similar to the console port, but available remotely through a modem. It's slowly disappearing.

Telnet and SSH

Telnet is a in-band method for remotely establishing a CLI session in a virtual terminal of the device.

Unlike the console connection, Telnet sessions require functional networking services (in-band) on the device.

The Secure Shell (SSH) protocol provides a remote login similar to Telnet, except that it uses proper security.

SSH provides stronger password authentication than Telnet and data are sent only after encryption.

IOS includes client and server software for both, so we can use SSH and Telnet to connect to and from Cisco devices.

Cisco CLI modes of operation

The IOS CLI uses a hierarchical structure for the modes. From most basic to most specialized, the major modes are:

  • User executive (User EXEC) mode
  • Privileged executive (Privileged EXEC) mode
  • Global configuration mode
  • Other specific configuration modes (e.g. interface, line).

Each mode has a distinctive prompt and is used to accomplish particular tasks with a specific set of commands available only into that mode.

GNU/Linux CLI is not modal, but Cisco IOS modes can be emulated by Quagga.

User EXEC Mode

The User EXEC mode is at the most basic level of the modal hierarchical structure, the first encountered upon entrance into the CLI of an IOS device.

It allows only a limited number of basic monitoring commands. It's often referred to as view-only mode.

The user EXEC mode is identified by the CLI prompt ending with the ">" symbol, for instance:

Switch>

Privileged EXEC Mode

The execution of configuration and management commands requires entering the privileged EXEC mode or a more specific mode in the hierarchy.

Of course User EXEC mode monitoring command are also available here.

The privileged EXEC mode can be identified by the prompt ending with the "#" symbol.

Switch#

Privileged EXEC Mode

Global configuration mode and all other more specific configuration modes can only be reached from the privileged EXEC mode.

Privileged EXEC mode does not require authentication by default, so it's considered best practice to configure it.

The enable and disable commands are used to switch the CLI between the user EXEC mode and the privileged EXEC mode, entering and leaving them respectively.

It's sometimes referred to as the enable mode.

Global Configuration Mode

From global configuration mode, CLI configuration changes are made that affect the operation of the device as a whole.

The following CLI command is used to take the device from privileged EXEC mode to the global configuration mode:

Switch# configure terminal

After the command is executed, to show that the devices is in global configuration mode, the prompt becomes

Switch(config)#

It's also shortened as global config.

Specific Configuration Modes

From the global config mode, the user can enter different sub-modes that allows for configuration of specific parts or functionalities of the device.

They're the CLI equivalent to the applet in the Windows Control Panel.

  • Interface mode - to configure a network interface.
    • Switch(config)# interface fa0/1
Switch(config-if)#
  • Line mode - to configure one of the physical or virtual lines.
    • Switch(config)# line console 0
Switch(config-line)#

Specific Configuration Modes

To leave a sub-mode and return to the previous mode, use the exit command; to exit completely and return to privileged EXEC mode, enter end or Ctrl-Z.

It is possible to switch directly from a sub-mode to another:

Switch(config-if)# line console 0
Switch(config-line)# interface S0/0/1
Switch(config-if)#

Structure of IOS Commands

Cisco IOS commands have a syntax composed of a command combined with several keywords and arguments.

  • The command is actually the first word, describing the major action to perform.
  • Keywords are pre-defined words that specialize the behaviour of the command.
  • Arguments are not pre-defined, entered by the user to provide informations to the command.

Keywords and arguments can be optional in some cases and required in others. All of this is detailed in the Cisco IOS Command Reference.

Structure of IOS Commands

Here an example of a keyword altering the behaviour of a command:

Switch# show running-config

Note: very often, for simplicity, the command + keyword(s) combination is referred to as a command, so we talk about the show running-config command.

Here's how user-submitted arguments can achieve the exact desired effect:

Switch(config-if)# description [string]
Switch(config-if)# description This is the Hacklab router
Switch> ping 10.10.10.5

Show Command Keyword(s)

Wow. Such Command. Much Keyword. Very Context.

Context-Sensitive Help

The context-sensitive help provides a list of commands and the arguments associated with those commands within the context of the current mode.

To access context-sensitive help on Cisco IOS, enter a question mark, "?", at any (point in the) prompt.

There's an immediate response without the need to Enter.

On Gnu/Linux press TAB for command completion, or use man command for the included documentation.

IOS CLI evaluates left-to-right. If a command is entered incorrectly (mispellings, missing arguments, etc.), a marker to the precise point its evaluation failed will appear.

IOS Hot Keys

  • Down Arrow - Allows the user to scroll forward through former commands.
  • Up Arrow - Allows the user to scroll backward through former commands.
  • Tab - Completes the remainder of a partially typed command or keyword.
  • Ctrl-A - Moves to the beginning of the line.
  • Ctrl-E - Moves to the end of the line.
  • Ctrl-R - Redisplays a line.
  • Ctrl-Z - Exits the config mode and returns to user EXEC.
  • Ctrl-C - Exits the config mode or aborts current command.
  • Ctrl-Shift-6 - Allows the user to interrupt an IOS process.

IOS Shortcuts

An IOS command can be entered in many shortened forms: as long as there's no ambiguity IOS will recognize it.

For instance, these are all accepted forms of the show running-config command:

Switch# show running
Switch# show run
Switch# sh ru

On the contrary, show processes can't be shortened to

Switch# show pro
Switch# show pr

because of the ambiguity with the show protocols command.

Device Hostname

Hostnames allow devices to be identified by network administrators over a network or the Internet.

By choosing the name wisely, it will become self-documenting. It helps to establish a naming convention.

In the global configuration mode, enter the hostname:

Switch(config)# hostname Sw-Floor-1

After the command is executed, the prompt will change to:

Sw-Floor-1 (config)#

To remove the name of a device, use:

Sw-Floor-1 (config)# no hostname

IOS Basic Security Checklist

  • Secure Privileged EXEC
  • Secure User EXEC
  • Secure Telnet
  • Encrypt passwords
  • Legal Disclaimer

Extremely important not to forget these tasks, both in the real world and for Cisco tests and exams.

Securing Privileged Access

To secure Privileged EXEC access, use:

Router(config)# enable secret mypassword

An older not encrypted variation of this command is the 

Router(config)#enable password mypassword

In Gnu/Linux many users can be created with different privileges. To change a user password:

passwd [name of the user]
Enter password:

Use a strong password: the longer the better; special characters; but most of all: don't reuse the same passwords and don't be predictable.

Secure User Access

To secure User EXEC means protecting the line used to access it, mostly the console port, with a password. Use:

Switch(config)# line console 0
Switch(config-line)# password cisco
Switch(config-line)# login
  • The zero in line console 0 represents the first (and almost always single) console interface.
  • The login command is mandatory, otherwise the password won't be prompted at the next access.

Securing VTY/Telnet

A virtual terminal is a virtual keyboard + screen combination which can be accessed over Telnet.

There are multiple VTYs (up to 16 in a typical Cisco device), numbered from 0 to 15. The exact number depends on the device and IOS version.

To set a password on vty lines use the following commands:

Switch(config)# line vty 0 15
Switch(config-line)# password cisco
Switch(config-line)# login

Encrypting Passwords

By default, passwords are stored unencrypted on IOS. Try:

Switch# show running-config

The service password-encryption command replaces clear-text passwords with weakly encrypted versions.

This encryption is only valid to passwords stored in config files, it doesn't protect passwords sent over media through unsecure protocols.

It's only useful to prevent unauthorized individuals from viewing passwords written in the configuration.

Linux stores the encrypted passwords in /etc/shadow.

Banner Messages

In many legal systems, unauthorized access is not prosecutable if there isn't an explicit notification that it's not allowed.

It is therefore vital to add a banner, a message that will be displayed in the device output after logging in.

In IOS, the best place for such legal disclaimers it's the message of the day (MOTD). It's displayed to all connected terminals, so choose careful wording.

Banner Messages

To configure a MOTD from global configuration mode:

Switch(config)# banner motd #
message #

The "#" is the limiting character. It could be any character. its only purpose is to signal the end of our MOTD.

Select a limiting character that's not supposed to appear in the text.

On Linux one way to do it is by editing the /etc/motd file.

Running ans Startup Config

There are 2 configuration files on Cisco IOS:

  • The running configuration is the currently active configuration of the device. It's stored in RAM.
  • The startup configuration is the configuration that will be loaded after each boot. It's stored in the NVRAM.

Changes are immediately active, and stored in the running config. But if not saved, they'll be lost at the next boot.

To make the changes permanent:

Switch# copy running-config startup-config

To restore a saved configuration the command is:

Switch# copy startup-config running-config

Config Backup and Restore

The IOS configuration can be saved on your PC using the capture capabilities of the terminal.

Putty, Minicom, and the other terminals have a log or capture function to get the output as text.

Enable logging, execute show running-config and save the output in a file.

You can restore the saved configuration on a device by copy-paste into the terminal emulator, but it will require some prior editing.

IP Addressing

The structure of an IPv4 address is called dotted decimal notation and is represented with four decimal numbers between 0 and 255.

IPv4 addresses are numbers assigned to individual devices connected to a network to uniquely identify them.

With the IP address, a subnet mask is also necessary.

A subnet mask is a special type of IPv4 address that, coupled with the IP address, determines which particular subnet of a larger network the device is a member.

For example:

  • IP: 192.168.1.1
  • Subnet mask: 255.255.255.0

Ethernet Basics

Ethernet is the most common local area network (LAN) technology used today.

Ethernet ports are found on end user devices, switch devices, and other networking devices that can physically connect to the network using a cable.

For a cable to connect devices using an Ethernet port, the cable must have the correct connector, an RJ-45.

Switch Virtual Interfaces

Cisco IOS switches have physical ports for devices to connect to, but also have one or more switch virtual interfaces (SVIs).

These are virtual interfaces, because there is no physical hardware on the device associated with it;

An SVI is created in software.

Each switch comes with one SVI appearing in the default configuration out-of-the-box.

The default SVI is interface VLAN1.

Configuring VLAN 1

To access the switch remotely, an IP address and a subnet mask must be configured on the SVI.

To configure the SVI the commands are:

  • (enter the interface configuration mode) 
    interface vlan 1
  • (set IP address) 
    ip address 192.168.10.2 255.255.255.0
  • (enable the interface)
    • no shutdown

PC Network Settings

In order for an end device to communicate over the network, it must be configured with the correct IP address information.

The IP address, Subnet Mask, Default Gateway and DNS server must be provided.

The default gateway address is the IP address of the router interface used for network traffic to exit the local network.

The DNS server is used to maintain the mapping between the IP addresses and names of various devices.

IP address information can be entered into the PC manually (static addressing), or using Dynamic Host Configuration Protocol (DHCP).

DHCP allows end devices to have IP information automatically configured.

Verifying Connectivity with Loopback

The ping command is used to verify the internal IP configuration on a local host.

This test is accomplished by using the ping command on a reserved address called the loopback.

The loopback address, 127.0.0.1, is defined by the TCP/IP protocol as a reserved address that routes packets back to the host.

The command is:

ping 127.0.0.1

Verify switch interfaces

Use the show ip interface brief command to verify the condition of the switch interfaces.

S1# show ip interface brief

FastEthernet0/2    unassigned      YES   manual   up       up

Testing end-to-end Connectivity

From both PCs ping the IP of the other PC.

From PC1:

ping 192.168.1.10

From PC2:

Ping the IP of Switch VLANs from PCs.

ping 192.168.1.1
ping 192.168.1.2

If all packets are received the network is correctly working.
