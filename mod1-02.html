Operating Systems

An OS is the interface between the hardware and the user.

Its main components are a kernel, which directly manages the hardware, and a shell, which is the interface provided to the user.

A shell can be graphical (GUI, Graphical User Interface) or text-based (CLI, Command-Line Interface).

The OS on embedded devices is usually called firmware. A firmware managing the functions of networking devices such as switches or routers is a network OS.

GUI vs CLI

GUIs are, of course, much easier and user-friendly for the average user. It requires less knowledge.

A network technician, administrator or designer it's not the average user :-). CLIs provides tangible advantages for expert users:

  • They are a standard, well-documented way of interacting with the OS, while GUIs have more variety.
  • A CLI is very lightweight, while a GUI it's a much more complex software (thus almost always less stable and slower).
  • GUIs don't always expose all of the feature set, while CLIs provide access to everything configurable.

Cisco IOS

Cisco IOS (Internetwork Operating System) is a term that encompasses a number of different OSes that run on various networking devices.

There are many distinct variations of Cisco IOS:

  • IOS for switches, routers, and other Cisco networking devices.
  • IOS numbered versions for a given Cisco networking device.
  • IOS feature sets providing distinct packages of features and services.

In this course, we'll mainly focus on Cisco IOS Release 15.x.

GNU/Linux Operating System

Gnu/Linux is a general purpose free operating system based on Linux Kernel.

The are other free operating systems based on many flavours of BSD kernel.

Different packaged OS are called distributions, like Fedora, Red Hat, Ubuntu, Debian, Arch and many more.

Some are for networking purpose, like: IPCop, PFSense, ZeroShell, CertOS, OpenWRT and many more.

Accessing an IOS Device

There are several ways to access the CLI environment. The most common methods are:

  • Console
  • Telnet or SSH
  • AUX port

The console port is a management port that provides out-of-band access to Cisco device.

Out-of-band access refers to access via a dedicated management channel that is used for device maintenance purposes only.

It's used for performing an initial configuration of the networking device and for recovering purposes

Telnet and SSH

Telnet is a in-band method for remotely establishing a CLI session of a device, through a virtual interface, over a network.

Unlike the console connection, Telnet sessions require active networking services on the device

The Secure Shell (SSH) protocol provides a remote login similar to Telnet, except that it uses more secure network services.

SSH provides stronger password authentication than Telnet and uses encryption when transporting session data

An older way to establish a CLI session remotely is via a telephone dialup connection using a modem connected to the auxiliary (AUX) port of a router

Terminal emultors

There are many terminal emulation programs available for connecting to a networking device either by a serial connection over a console port or by a Telnet/SSH connection.

Some of these include:

  • PuTTY (Open Source)
  • Tera Term (Open Source)
  • SecureCRT
  • HyperTerminal
  • OS X Terminal
  • Minicom (Open Source)

Cisco CLI modes of operation

The CLI uses a hierarchical structure for the modes..

In hierarchical order from most basic to most specialized, the major modes are:

  • User executive (User EXEC) mode
  • Privileged executive (Privileged EXEC) mode
  • Global configuration mode
  • Other specific configuration modes, such as interface configuration mode.

Each mode has a distinctive prompt and is used to accomplish particular tasks with a specific set of commands that are available only to that mode.

On GNU/Linux modes can be emulated by Quagga software.

User EXEC Mode

The user EXEC mode is at the most basic level of the modal hierarchical structure.

This mode is the first mode encountered upon entrance into the CLI of an IOS device.

The user EXEC mode allows only a limited number of basic monitoring commands.

This is often referred to as view-only mode.

The user EXEC mode is identified by the CLI prompt that ends with the ">" symbol:


						Switch>

Privileged EXEC Mode (1/2)

The execution of configuration and management commands requires that the network administrator use the privileged EXEC mode or a more specific mode in the hierarchy.

The privileged EXEC mode can be identified by the prompt ending with the "#" symbol.


							Switch#

Privileged EXEC Mode (2/2)

Global configuration mode and all other more specific configuration modes can only be reached from the privileged EXEC mode.

By default, privileged EXEC mode does not require authentication. It is a good practice to ensure that authentication is configured.

The enable and disable commands are used to change the CLI between the user EXEC mode and the privileged EXEC mode, respectively.

Global Configuration Mode

From global configuration mode, CLI configuration changes are made that affect the operation of the device as a whole.

The following CLI command is used to take the device from privileged EXEC mode to the global configuration mode:


						Switch# configure terminal

After the command is executed, the prompt changes to show that the switch is in global configuration mode.


						Switch(config)#

See examples on Packet Tracer.

Specific Configuration Modes

From the global configuration mode, the user can enter different sub-configuration modes. For example:

  • Interface mode - to configure one of the network interfaces (Fa0/0, S0/0/0)
  • Line mode - to configure one of the physical or virtual lines (console, AUX, VTY)

To exit a specific configuration mode and return to global configuration mode, enter exit at a prompt.

To leave configuration mode completely and return to privileged EXEC mode, enter end or use the key sequence Ctrl-Z.

Basic IOS Commands

The show command is used to display information about the device.

This command has various keywords that must be used to define what particular output should be displayed.

For example the following command specifies that the running configuration is to be displayed as the output:


						Switch# show running-config

A command might require one or more arguments. Unlike a keyword, an argument is generally not a predefined word. An argument is a value or variable defined by the user.

For instance the syntax for using the description command is:


						Switch(config-if)# description string
					

						Switch> ping 10.10.10.5

Context-Sensitive Help

The context-sensitive help provides a list of commands and the arguments associated with those commands within the context of the current mode.

To access context-sensitive help on Cisco IOS, enter a question mark, ?, at any prompt.

There is an immediate response without the need to use the Enter key.

On Gnu/Linux press TAB for command completion, or use man command for a detailed guide for a command

Try them on Packet tracer and Linux console.

IOS Hot Keys and Shortcuts (1/2)

  • Down Arrow - Allows the user to scroll forward through former commands.
  • Up Arrow - Allows the user to scroll backward through former commands.
  • Tab - Completes the remainder of a partially typed command or keyword.

IOS Hot Keys and Shortcuts (2/2)

  • Ctrl-A - Moves to the beginning of the line.
  • Ctrl-E - Moves to the end of the line.
  • Ctrl-R - Redisplays a line.
  • Ctrl-Z - Exits the configuration mode and returns to user EXEC.
  • Ctrl-C - Exits the configuration mode or aborts the current command.
  • Ctrl-Shift-6 - Allows the user to interrupt an IOS process such as ping or traceroute.

Device Hostname

Hostnames allow devices to be identified by network administrators over a network or the Internet.

Define a naming convention for all the network machines. For example if we have three switches at different floors, name it as Sw-Floor-1, Sw-Floor-2, and Sw-Floor-3.

in the global configuration mode, enter the hostname:


						Switch(config)# hostname Sw-Floor-1

After the command is executed, the prompt will change to:


						Sw-Floor-1 (config)#

To remove the name of a device, use:


						Sw-Floor-1 (config)# no hostname

Securing privileged access

To secure privileged EXEC access, use:


						enable secret mypassword

An older not encrypted variation of this command is the 


						enable password mypassword

Gnu/Linux is a general purpose multi-user Operating System. Many users can be created with different privileges.

To change the password of a user use passwd command:

passwd [name of the user]

Secure console access

To set a password for the console line use the following commands in global configuration mode:


						Switch(config)# line console 0
					

						Switch(config-line)# password cisco
					

						Switch(config-line)# login

The command line console 0 is used to enter line configuration mode for the console. The zero is used to represent the first (and in most cases only) console interface.

VTY Password

The vty lines allow access to a Cisco device via Telnet.

By default, many Cisco switches support up to 16 vty lines that are numbered 0 to 15.

The number of vty lines supported on a Cisco router varies with the type of router and the IOS version.

However, 5 is the most common number of vty lines configured.

To set a password on vty lines use the following commands:


						Switch(config)# line vty 0 15
					

						Switch(config-line)# password cisco
					

						Switch(config-line)# login

Encrypting Passwords

By default passwords are unencrypted. Try:


						Switch# show running-config

The service password-encryption command applies weak encryption to all unencrypted passwords.

This encryption applies only to passwords in the configuration file, not to passwords as they are sent over media.

The purpose of this command is to keep unauthorized individuals from viewing passwords in the configuration file.

Run show running-config again and see the difference.

In Linux the encrypted passwords are stored in /etc/shadow file.

Banner Messages

It is vital to provide a method for declaring that only authorized personnel should attempt to gain entry into the device. To do this, add a banner to the device output.

The IOS provides multiple types of banners. One common banner is the message of the day (MOTD). It is often used for legal notification because it is displayed to all connected terminals.

To configure a MOTD from global configuration mode:

Switch(config)# banner motd #
message #

On Linux there are many ways, one is to edit the file /etc/motd.

Save configuration

The running configuration file is stored in the working memory of the device, or random access memory (RAM).

If the device is restarted, all configuration changes will be lost unless they have been saved.

The startup configuration file reflects the configuration that will be used by the device upon reboot.

The startup configuration file is stored in NVRAM.

The command to save the running configuration to startup configuration file is:

Switch# copy running-config startup-config

To restore a saved configuration the command is:

Switch# copy startup-config running-config

Delete configuration

The startup configuration is removed on both Switches and Routers by using

Switch# erase startup-config

On a switch you must also issue the command

Switch# delete vlan.dat

Backup configuration via console

The IOS configuration can be saved on your PC using the capture capabilities of the terminal.

HyperTerminal, Minicom, Tera term and the other terminals have a log or capture function to get the output as text.

Enable logging, execute a show running-config command and save the output in a file.

You can edit the file and restore it on the Switch or Router.

IP Addressing

The structure of an IPv4 address is called dotted decimal notation and is represented with four decimal numbers between 0 and 255.

IPv4 addresses are numbers assigned to individual devices connected to a network to uniquely identify them.

With the IP address, a subnet mask is also necessary.

A subnet mask is a special type of IPv4 address that, coupled with the IP address, determines which particular subnet of a larger network the device is a member.

For example:

  • IP: 192.168.1.1
  • Subnet mask: 255.255.255.0

Ethernet Basics

Ethernet is the most common local area network (LAN) technology used today.

Ethernet ports are found on end user devices, switch devices, and other networking devices that can physically connect to the network using a cable.

For a cable to connect devices using an Ethernet port, the cable must have the correct connector, an RJ-45.

Switch Virtual Interfaces

Cisco IOS switches have physical ports for devices to connect to, but also have one or more switch virtual interfaces (SVIs).

These are virtual interfaces, because there is no physical hardware on the device associated with it;

An SVI is created in software.

Each switch comes with one SVI appearing in the default configuration out-of-the-box.

The default SVI is interface VLAN1.

Configuring VLAN 1

To access the switch remotely, an IP address and a subnet mask must be configured on the SVI.

To configure the SVI the commands are:

  • (enter the interface configuration mode) 
    interface vlan 1
  • (set IP address) 
    ip address 192.168.10.2 255.255.255.0
  • (enable the interface)
    • no shutdown

PC Network Settings

In order for an end device to communicate over the network, it must be configured with the correct IP address information.

The IP address, Subnet Mask, Default Gateway and DNS server must be provided.

The default gateway address is the IP address of the router interface used for network traffic to exit the local network.

The DNS server is used to maintain the mapping between the IP addresses and names of various devices.

IP address information can be entered into the PC manually (static addressing), or using Dynamic Host Configuration Protocol (DHCP).

DHCP allows end devices to have IP information automatically configured.

Verifying Connectivity with Loopback

The ping command is used to verify the internal IP configuration on a local host.

This test is accomplished by using the ping command on a reserved address called the loopback.

The loopback address, 127.0.0.1, is defined by the TCP/IP protocol as a reserved address that routes packets back to the host.

The command is:

ping 127.0.0.1

Verify switch interfaces

Use the show ip interface brief command to verify the condition of the switch interfaces.
