Small Business' Networks

The knowledge required to run a small network is similar to the one for a larger network. It is the number of devices involved that changes.

Small network typically involve a single router, a single WAN connection, one or more switches and APs (often integrated in one device).

The management can be done by an employee or a contractor.

Factors in Selecting a Device

  • Cost: capacity, features, available ports, type of ports, backplane speed, they all influence how expensive a device is. But there are device-independent costs like:
    • Cable Runs: the devices have to be connected together.
    • Redundancy: no matter which devices are selected, redundancy requires more of them.
  • Speed/Type/Number of Ports: We have to decide whether to choose based on present or future needs also.
    • Expandability: additional modules offer flexibility, but you still have to think about your needs.
  • Software Features: has the NOS every functionality I need from it? VPNs, QoS, routing protocols, NAT, etc.

Redundancy

Redundancy provides networks with reliability.

It is achieved by duplicating: devices, links, power supplies, services.

Redundancy can be introduces progressively, starting with a second connection between multiple intermediary devices.

Small networks have typically one gateway to the Internet. A backup connection to a second ISP has to be considered.

Small Network Design Checklist

  1. Create centralized locations for network resources and intermediary devices.
  2. Secure unauthorized physical access.
  3. Secure unauthorized logical access.
  4. Create physical redundancy.
  5. Create logical redundancy.
  6. Understand the traffic flowing in the network in order to implement necessary priorities.

Software in a Small Network

Software that makes use of the network can be divided in:

  • Network Applications: these are the applications that know how to use the network directly (network-aware) because they implement L7 protocols themselves (email clients and browsers are common examples).
  • Network Services: these are stand-alone Application Layer protocols implementations that can be used by other applications that need to send data over the network. Print spooling, network file systems are good examples of L7 Services.

Real-Time Applications

Real-time applications are the ones that stress the most the need for a good network design.

They rely on priority delivery, which needs careful planning, a capable infrastructure, proper equipment and configurations.

Typical real-time applications in a small network are Voice Over IP (VoIP) and IP Telephony. The difference stands in whether the phone performs the voice-to-IP conversion or not.

However, VoIP is the term universally used.

Real-Time applications need protocols that supports delay-sensitive delivery and QoS, like the Real-Time Transport Protocol (RTP) or the Real-Time Transport Control Protocol (RTCP).

Often QoS is implemented in-hardware.

From Small to Larger Network

Networks can grow either reactively (responding to needs) or proactively (anticipating needs). It is clearly better to grow the network alongside its needs.

To carefully plan the scaling of a network you need: documentation, inventory, budget, traffic analysis.

Traffic/Protocol analysis allows an administrator to collect statistics about the traffic flowing in the network. With the right data, optimizations can be made. Some suggestions:

  • Analyze the traffic in the peak hours.
  • Analyze traffic globally, but also in each specific segment.
  • Document employee-snapshots over time.

Analysis could reveal optimization paths that are trivial, or others requiring major overhaul of the network design.

Types of Network Threats

  • Information Theft: information is obtained that was supposed to be confidential and that is now out of control and can be used or sold or otherwise damage the original owner.
  • Identity Theft: like the above, but the information allows the attacker to assume someone else's identity in order to achieve access or other type of unauthorized usage.
  • Data Loss or Tampering: computer data that is purposefully damaged to harm the owener of the data, or altered to obtain an advantage.
  • Disruption of Service: even without breaking into a system, attackers can impede legitimate users of a service to access it.

Physical Security

Device physical protection is as important as protecting the software stack. The Attacker can be an outsider, but we can also play our (negative) part in these categories of threats:

  • Hardware: intentional physical damage to hosts or links.
  • Environmental: making a device operate outside of its temperature window.
  • Electrical: over/under-voltage, partial/unstable/total power loss, incorrect grounding, electrostatic discharge.
  • Maintenance: no labeling, poor cabling, lack of spares.

Vulnerabilities

Vulnerabilities can be found in:

  • Technology - Protocols, software, operating systems, they all have flaws which can be exploited.
  • Configuration - Unsecured, unused user accounts; weak passwords; unused and unsecured software and services; weak default settings; general misconfigurations.
  • Policy - Unwritten security policies; politics; unapplied security policies; inadequate monitoring; unauthorized and undocumented changes; lack of a recovery plan.

Network Attacks - Malicious Code

There are programs created for the purpose of causing harm:

  • Viruses - software included in another programs to perform malicious functions. Often human interactions is required.
  • Worms - Stand-alone applications written to exploit targeted vulnerabilities. They don't need human interaction.
  • Trojan Horses - Applications that are written to appear normal, while performing attacks on the system.

In order to be executed, malicious code attacks need:

  • Some vulnerability to exploit, whether software or human.
  • A propagation mechanism to spread themselves.
  • The actual payload to propagate and execute.

Network Attacks - Reconnaissance

Reconnaissance (ricognizione, in Italian) involves the search, discovery and mapping of vulnerabilities in victim's systems.

  • Queries - Tools like nslookup or whois can be used to determine the address space belonging to the victim.
  • Mass Pings - The attacker can use tools to automatically ping large block of addresses to discover which one are active.
  • Port Scanning - Using automated tools like nmap an attacker can discover the transport layer ports that are opened on a system.
  • Packet Sniffing - If an attacker takes control of a system along the traffic paths, He can monitor traffic using software like Wireshark.

Network Attacks - Access

Access attacks consist of unauthorized (attempted or successful) access to systems and exploitation of user privileges.

  • Password Attacks - They can be performed with trojan horses, packet sniffers or as brute force attacks.
  • Trust Exploitation - Instead of attacking a system directly, the attacker takes control of a system or service (e.g. Active Directory, or NFS) trusted by the victim.
  • Port Redirection - A cas of trust exploitation where an attacker compromises an host to bypass firewall rules that drops direct attacks (see the next slides).
  • Man-In-The-Middle - The attacker acquires the ability to monitor a comunication channel, altering traffic and injecting messages.

Network Attacks - Denial of Service

There are many types of DoS attacks, but they all have the purpose of exhausting system and network resources to the point where it's hard or impossible to legitimately use them.

Some kinds of Denial of Service attacks are:

  • Ping of Death - Malformed, large (more than IP max size) ICMP packets that are meant to crash receiving hosts.
  • SYN Flood - The attacker begins (SYN) a lot of TCP 3-Way Handshakes without ever completing them (SYN ACK).
  • Smurf Attack - The attacker directs large amount of traffic to a victim by using the spoofed address as the source, and a broadcast address as the destination of malformed packets.
  • Distributed Denial of Service (DDoS) - When the attacker has a large number of compromised systems (Handlers and Zombies) at his disposal, simultaneously targeting a victim.

Network Attack Mitigation

If an attack is successful, administrators have to perform these steps in order to regain control: containment, inoculation (immunization), quarantine, treatment.

A central patch server, automatic updates, and a standard software image can be good strategies to keep systems secure.

The Triple A governs access control by providing:

  • Authentication - Who is permitted to access the network. Local authentication, like authentication commands on IOS, doesn't scale. External authentication like RADIUS or TACACS+, is recommended for large networks.
  • Authorization - What authorized users can or cannot do.
  • Accounting - Keeping tracks (logging) of accessed resources and users' activity on a network.

Firewalls

Firewalls are systems placed between networks, controlling the traffic flow between them using several techniques:

  • Packet Filtering based on L2 or L3 addresses.
  • Application Filtering based on port numbers and associated applications.
  • URL filtering targeted at specific websites or keywords.
  • Stateful Packet Inspection (SPI) can recognize if incoming packets are responses or unsolicited traffic.
  • Network Address Translation is often performed by firewalls, and it allows to hide internal IP addresses.

Firewall can take differents forms, such as appliances (dedicated hardware), server software (running on a full OS), integrated (in multi-purpose devices) or personal firewalls (on each host).

Basic Security Practices

  • Don't rely on default settings being secure, review them!
  • Change default username and passwords.
  • Principle of Least Priviledge: configure users and permissions to provide access only to the minimum extent necessary.
  • Least Attack Surface: uninstall or disable unused services.
  • New device != up-to-date device. Patch first!
  • The longer the password, the better. Use passphrases.
  • Avoid dictionary words, common or easily guessable patterns.
  • Entropy is good: mispell the passwords, mix the letter case, symbols, numbers.
  • Change the password often to reduce opportunity window and do not write it down.

Network Baseline

Establishing a network baseline means detailing network performances over time to create a reference point.

Basic performance tests like ping or traceroute should be executed at different times of the day, days of the week, and stored as documentation for future comparisons.

There are integrated softwares that manage the creation and storage of large and detailed baselines.

Cisco Discovery Protocol

CDP is a proprietary protocol by Cisco that allows a device to learn hardware and software information about other devices directly connected to it.

CDP runs at L2, so it can operate even if there is no network layer connectivity or different L3 protocols are running.

It provides details like Device identifiers, address list, port identifiers, capabilities, platform.

CDP is enabled by default on Cisco IOS. It can be disabled globally with the command no cdp run, or per-interface with the command no cdp enable.

Cisco IOS File System (IFS)

The Cisco IOS File System is a unified interface to every file system that is available on a switch or a router.

Cisco IFS allows an administrator to operate on directory and files seamlessly, even when the file system is remote.

Cisco IOS supports standard UNIX commands to operate on files, like cd, pwd or dir.

The command show file systems list all available file system with their prefixes and permissions (e.g. ro/rw/wo)

Wireless Settings and Security

  • Wireless Mode - It's the wireless standard revision (802.11a/b/g/n/ac) that will be used. Many routers support mixed mode, and the standard themselves are backwards-compatible, but it can result in reduced performances.
  • Service Set Identifier (SSID) - Alphanumeric, case-sensitive, unique name for a WLAN, up to 32 chars.
  • Channel - The exact frequency range that the communication over that WLAN will use. There are multiple channels so that many WLANs can works in proximity.
  • SSID Broadcast - As a security feature, WLAN names can be hidden. Only those who know its value can then connect to it.
  • Encryption - WEP and WPA/WPA2 are protocols to encrypt WiFi data. WEP is old and very easily crackable, so you MUST use WPA (that uses dynamically-generated keys).
